Module 4: Natural Language Processing

What Is NLP?

Natural Language Processing (NLP) is the branch of AI that deals with understanding, interpreting, and generating human language. It powers the tools you use every day — ChatGPT, Google Translate, email spam filters, autocomplete on your phone, and sentiment analysis in customer reviews.

The core challenge: computers work with numbers, but language is ambiguous, contextual, and full of nuance. "I saw a bat" means something completely different in a sports shop versus a cave. NLP bridges this gap.

Text Preprocessing Pipeline

Before feeding text to any model, you need to clean and normalise it. Raw text from the real world contains noise that will confuse your algorithm. The standard pipeline:

  1. Lowercasing — "Python" and "python" should be the same word
  2. Tokenisation — split text into individual words or sub-words
  3. Stop word removal — remove common words like "the", "is", "a" that carry little meaning
  4. Stemming / Lemmatisation — reduce words to their base form
  5. Removing punctuation and special characters — strip symbols that add noise
Stemming vs Lemmatisation

Stemming is fast but crude — it chops off word endings using simple rules. "Running", "runner", "ran" all become "run" (or sometimes "runn"). Lemmatisation uses a real dictionary and grammar rules to return the proper base form. "Better" → "good". Always prefer lemmatisation when accuracy matters.

Coding Basics: Strings, Lists, and Filtering

Text starts as a string, which is how code stores words and sentences. NLP usually turns that string into a list of smaller pieces called tokens.

Filtering is a core coding skill: keep what matters and remove what does not. In NLP, that means removing punctuation or common words so the model can focus on meaningful language.

Reference: The Full NLTK Pipeline

This is the production-grade version — it uses NLTK's real tokeniser, stop word list, and lemmatiser, which requires downloading NLTK data. Run this in Colab or Jupyter, not in the browser sandbox.

import nltk
from nltk.tokenize import word_tokenize
from nltk.corpus import stopwords
from nltk.stem import PorterStemmer, WordNetLemmatizer
import re

# Download required NLTK data (only needed once)
nltk.download('punkt', quiet=True)
nltk.download('stopwords', quiet=True)
nltk.download('wordnet', quiet=True)
nltk.download('punkt_tab', quiet=True)

def preprocess(text, use_lemmatization=True):
    """Full NLP preprocessing pipeline."""

    # Step 1: Lowercase
    text = text.lower()

    # Step 2: Remove punctuation and numbers
    text = re.sub(r'[^a-z\s]', '', text)

    # Step 3: Tokenise — split into individual words
    tokens = word_tokenize(text)

    # Step 4: Remove stop words
    stop_words = set(stopwords.words('english'))
    tokens = [t for t in tokens if t not in stop_words]

    # Step 5: Lemmatise or stem
    if use_lemmatization:
        lemmatizer = WordNetLemmatizer()
        tokens = [lemmatizer.lemmatize(t) for t in tokens]
    else:
        stemmer = PorterStemmer()
        tokens = [stemmer.stem(t) for t in tokens]

    return tokens

# ── Test it ───────────────────────────────────────────────────────────
samples = [
    "The quick brown foxes are jumping over the lazy dogs!",
    "Machine Learning algorithms are learning from historical data.",
    "She was running faster than the other runners in the competition."
]

for text in samples:
    print(f"Original:  {text}")
    print(f"Processed: {preprocess(text)}")
    print()

Representing Text as Numbers

Once text is cleaned, you need to convert it to numerical form. There are two main approaches, each with different trade-offs:

Bag of Words / TF-IDF

Bag of Words counts how many times each word appears in a document. It completely ignores word order and context — "the cat sat on the mat" and "the mat sat on the cat" are identical. Simple but surprisingly effective for many tasks.

TF-IDF (Term Frequency–Inverse Document Frequency) improves on this by downweighting common words that appear in many documents (like "the") and upweighting rare, distinctive words. It's the standard for text classification and search.

Step 2 — Vectorise with TF-IDF & Train

Convert each review into a vector of TF-IDF scores, then train a Logistic Regression classifier on those vectors.

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report

vectorizer = TfidfVectorizer(max_features=1000, stop_words='english')
X_train_vec = vectorizer.fit_transform(X_train)
X_test_vec  = vectorizer.transform(X_test)

print(f"Vocabulary size: {len(vectorizer.vocabulary_)}")
print(f"Feature matrix shape: {X_train_vec.shape}")

clf = LogisticRegression(random_state=42)
clf.fit(X_train_vec, y_train)

y_pred = clf.predict(X_test_vec)
print("\n=== Classification Report ===")
print(classification_report(y_test, y_pred,
                             target_names=['Negative', 'Positive']))

Word Embeddings

TF-IDF treats every word as completely independent. But "king" and "queen" are related; "cat" and "dog" are similar. Word embeddings capture these relationships by representing each word as a dense numerical vector in a high-dimensional space where similar words are close together.

The famous example: vector("king") - vector("man") + vector("woman") ≈ vector("queen"). The model has learned gender as a direction in vector space.

Word2Vec and GloVe are classic embedding models. Modern systems use contextual embeddings from Transformers (see below), where the same word gets a different vector depending on context.

Introduction to Transformers and BERT

The Transformer architecture (introduced in the 2017 paper "Attention Is All You Need") is the foundation of every modern NLP model — including GPT, BERT, and ChatGPT.

Its key innovation is the self-attention mechanism: instead of processing words one by one like older RNN-based models, Transformers look at the entire sentence at once and compute how much each word should "pay attention" to every other word. This captures long-range dependencies naturally.

BERT (Bidirectional Encoder Representations from Transformers) pre-trains on billions of words and can be fine-tuned for specific tasks in minutes. It is the backbone of Google Search.

from transformers import pipeline

sentiment = pipeline(
    "sentiment-analysis",
    model="distilbert-base-uncased-finetuned-sst-2-english"
)

# ── Analyse some texts ────────────────────────────────────────────────
texts = [
    "I absolutely love this course. The explanations are crystal clear!",
    "This was a complete waste of time. I learned nothing.",
    "It was okay. Some parts were good, others not so much.",
    "The deep learning module blew my mind. Highly recommend.",
    "Confusing explanations and poor examples."
]

results = sentiment(texts)

for text, result in zip(texts, results):
    label = result['label']
    score = result['score']
    print(f"[{label:8s} {score:.0%}] {text[:55]}...")
