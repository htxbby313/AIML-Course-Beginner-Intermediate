Module 5: Computer Vision

How Computers See Images

A digital image is simply a grid of pixels. Each pixel is a tiny square of colour. For a standard colour image in the RGB model, every pixel is represented by three numbers — one each for Red, Green, and Blue — each ranging from 0 to 255.

A 256×256 RGB image is therefore a 3D array of shape (256, 256, 3) containing 196,608 numbers. A computer vision model's job is to learn which patterns in those numbers correspond to meaningful objects.

  • Greyscale image: shape (height, width) — one channel, brightness only
  • RGB image: shape (height, width, 3) — three channels
  • RGBA image: shape (height, width, 4) — RGB plus transparency
Why Normalise Pixel Values?

Neural networks train far more stably when inputs are small numbers close to zero. Dividing pixel values by 255 scales them from [0, 255] to [0.0, 1.0]. This is the single most common image preprocessing step — always do it.

Coding Basics: Images Are Numbers

To a computer, an image is not a picture first. It is a grid of numbers. Each pixel has values that describe color and brightness.

This matters for AI because computer vision models learn from those numbers. Code is the bridge between what you see on screen and what the model receives as input.

Reference: Building & Inspecting an Image From Scratch

A more complete walkthrough using NumPy, Pillow, and Matplotlib — copy this for your own local environment if you want to go further than the pixel example above.

import numpy as np
from PIL import Image
import matplotlib
matplotlib.use('Agg')  # use non-interactive backend
import matplotlib.pyplot as plt

# ── Create a synthetic image from scratch ─────────────────────────────
# Let's build a simple 64x64 RGB image with a coloured gradient.
height, width = 64, 64
img_array = np.zeros((height, width, 3), dtype=np.uint8)

for row in range(height):
    for col in range(width):
        img_array[row, col, 0] = int(col / width * 255)   # Red   increases left→right
        img_array[row, col, 1] = int(row / height * 255)  # Green increases top→bottom
        img_array[row, col, 2] = 128                       # Blue  constant

print(f"Image shape: {img_array.shape}")           # (64, 64, 3)
print(f"Pixel at (0,0): {img_array[0, 0]}")       # top-left corner
print(f"Pixel at (32,32): {img_array[32, 32]}")   # centre

# ── Normalise ─────────────────────────────────────────────────────────
img_normalised = img_array.astype(np.float32) / 255.0
print(f"\nNormalised pixel at (32,32): {img_normalised[32, 32].round(3)}")
print(f"Min value: {img_normalised.min():.1f}, Max value: {img_normalised.max():.1f}")

# ── Channel operations ────────────────────────────────────────────────
red_channel   = img_array[:, :, 0]   # shape: (64, 64)
green_channel = img_array[:, :, 1]
blue_channel  = img_array[:, :, 2]

print(f"\nRed channel shape: {red_channel.shape}")
print(f"Red channel mean: {red_channel.mean():.1f}")

# ── Convert to greyscale ──────────────────────────────────────────────
# Standard luminance formula: weighted average of R, G, B
greyscale = (0.299 * img_array[:,:,0]
           + 0.587 * img_array[:,:,1]
           + 0.114 * img_array[:,:,2]).astype(np.uint8)
print(f"\nGreyscale shape: {greyscale.shape}")     # (64, 64) — no channel dim

Image Preprocessing for ML

Before feeding images into a model, you almost always need to preprocess them. The standard steps are:

  • Resizing — models expect fixed input dimensions. A CNN trained on 224×224 images cannot accept a 1920×1080 photo without resizing first.
  • Normalisation — divide pixel values by 255 (or use the dataset's mean/std for more precise normalisation).
  • Data Augmentation — artificially expand your training set by applying random transformations: horizontal flips, rotations, zooms, brightness changes. This helps the model generalise and dramatically reduces overfitting.

Reference: Data Augmentation Pipeline

Requires TensorFlow — run this in Colab or Jupyter, not in the browser sandbox.

import tensorflow as tf
from tensorflow.keras import layers
import numpy as np

# ── Define an augmentation pipeline ──────────────────────────────────
# In Keras, augmentation layers can be stacked inside the model itself.
# They only apply during training — not at inference time.
augmentation = tf.keras.Sequential([
    layers.RandomFlip("horizontal"),          # 50% chance of horizontal flip
    layers.RandomRotation(0.1),               # rotate up to ±36 degrees
    layers.RandomZoom(0.1),                   # zoom in/out by up to 10%
    layers.RandomTranslation(0.1, 0.1),       # shift by up to 10% in x/y
    layers.RandomContrast(0.2),               # adjust contrast randomly
], name="augmentation")

# ── Simulate applying augmentation to a batch of images ──────────────
# In practice you'd load real images; here we use random noise.
batch_of_images = tf.random.uniform(shape=(8, 224, 224, 3))

# Note: augmentation layers expect a 4D tensor (batch, H, W, C)
augmented_batch = augmentation(batch_of_images, training=True)

print(f"Input shape:  {batch_of_images.shape}")
print(f"Output shape: {augmented_batch.shape}")  # same shape, different values
print("Augmentation pipeline built successfully.")

# ── Alternative: tf.data pipeline (production-ready) ──────────────────
def augment_fn(image, label):
    """Apply augmentation to one image-label pair."""
    image = tf.image.random_flip_left_right(image)
    image = tf.image.random_brightness(image, max_delta=0.2)
    image = tf.image.random_contrast(image, lower=0.8, upper=1.2)
    image = tf.clip_by_value(image, 0.0, 1.0)
    return image, label

# Usage: dataset = dataset.map(augment_fn).batch(32).prefetch(tf.data.AUTOTUNE)
print("tf.data augmentation function defined.")

Image Classification with CNNs

The most common computer vision task is image classification: given an image, predict which of N categories it belongs to. CNNs (covered in Module 3) are the standard architecture for this.

Here we build a complete image classifier trained on CIFAR-10 — a dataset of 60,000 colour images across 10 classes (aeroplanes, cars, birds, cats, deer, dogs, frogs, horses, ships, trucks).

Reference: CIFAR-10 Image Classifier

Requires TensorFlow and downloads the CIFAR-10 dataset — run this in Colab or Jupyter, not in the browser sandbox.

import tensorflow as tf
from tensorflow.keras import layers, models
from tensorflow.keras.datasets import cifar10
import numpy as np

# ── Load CIFAR-10 ─────────────────────────────────────────────────────
(X_train, y_train), (X_test, y_test) = cifar10.load_data()

# Normalise pixel values to [0, 1]
X_train = X_train.astype('float32') / 255.0
X_test  = X_test.astype('float32')  / 255.0

CLASS_NAMES = ['airplane','automobile','bird','cat','deer',
               'dog','frog','horse','ship','truck']

print(f"Training set: {X_train.shape}  Labels: {y_train.shape}")
print(f"Test set:     {X_test.shape}")
print(f"Classes: {CLASS_NAMES}")

# ── Build the CNN with augmentation ──────────────────────────────────
model = models.Sequential([
    # Data augmentation (only active during training)
    layers.RandomFlip("horizontal"),
    layers.RandomRotation(0.1),
    layers.RandomZoom(0.1),

    # First convolutional block
    layers.Conv2D(32, (3,3), activation='relu', padding='same',
                  input_shape=(32,32,3)),
    layers.BatchNormalization(),
    layers.Conv2D(32, (3,3), activation='relu', padding='same'),
    layers.MaxPooling2D(2,2),
    layers.Dropout(0.25),

    # Second convolutional block (more filters = more complex features)
    layers.Conv2D(64, (3,3), activation='relu', padding='same'),
    layers.BatchNormalization(),
    layers.Conv2D(64, (3,3), activation='relu', padding='same'),
    layers.MaxPooling2D(2,2),
    layers.Dropout(0.25),

    # Third convolutional block
    layers.Conv2D(128, (3,3), activation='relu', padding='same'),
    layers.BatchNormalization(),
    layers.MaxPooling2D(2,2),
    layers.Dropout(0.4),

    # Classification head
    layers.Flatten(),
    layers.Dense(256, activation='relu'),
    layers.Dropout(0.5),
    layers.Dense(10, activation='softmax')  # 10 CIFAR-10 classes
])

model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=1e-3),
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

print(f"\nTotal parameters: {model.count_params():,}")

# ── Train (reduce epochs for quick testing) ───────────────────────────
history = model.fit(
    X_train, y_train,
    epochs=20,                  # increase to 50+ for better accuracy
    batch_size=64,
    validation_split=0.1,
    verbose=1
)

# ── Evaluate ──────────────────────────────────────────────────────────
loss, acc = model.evaluate(X_test, y_test, verbose=0)
print(f"\nTest Accuracy: {acc:.1%}")  # expect ~75-80% after 20 epochs

# ── Predict on test images ────────────────────────────────────────────
predictions = model.predict(X_test[:5], verbose=0)
for i, pred in enumerate(predictions):
    predicted_class = CLASS_NAMES[np.argmax(pred)]
    actual_class    = CLASS_NAMES[int(y_test[i])]
    confidence      = np.max(pred)
    print(f"Image {i+1}: Predicted={predicted_class:12s} "
          f"Actual={actual_class:12s} Confidence={confidence:.1%}")

Object Detection and Image Segmentation

Classification tells you what is in an image. Object detection also tells you where — it draws a bounding box around each detected object and labels it. Image segmentation goes further, classifying every single pixel.

  • Object Detection — outputs bounding boxes + class labels. Used in self-driving cars (detecting pedestrians, traffic lights), security cameras, retail analytics. Popular models: YOLO (real-time), Faster R-CNN (accurate).
  • Semantic Segmentation — assigns a class to every pixel (e.g. sky, road, car, person). Used in medical imaging, autonomous driving. Popular models: U-Net, DeepLab.
  • Instance Segmentation — like semantic segmentation but distinguishes between individual instances (car #1 vs car #2). Mask R-CNN is the standard.
