Module 1: AI/ML Foundations

What Is Artificial Intelligence?

Artificial Intelligence (AI) is a broad field of computer science focused on building systems that can perform tasks that normally require human intelligence — things like understanding language, recognising images, making decisions, and learning from experience.

AI is not one single technology. It is a collection of techniques and approaches. You encounter AI every day: when Netflix recommends a show, when your phone unlocks with your face, or when a spam filter catches a phishing email. All of these are AI at work.

Key Insight

Think of AI, ML, and Deep Learning as a set of nested circles. AI is the outermost circle (the big idea). Machine Learning sits inside it (one way to achieve AI). Deep Learning sits inside ML (a powerful subset of ML techniques).

Machine Learning (ML)

Machine Learning is the most widely used approach to building AI today. Instead of a programmer writing explicit rules ("if the email contains the word 'winner', mark it as spam"), an ML system learns the rules itself by examining thousands of examples.

The core idea: given enough data, an algorithm can discover patterns that a human might never think to code manually. This is why ML has transformed fields from medicine to finance to entertainment.

The Three Types of Machine Learning

1. Supervised Learning

The model is trained on a labelled dataset — every example has a known correct answer. The model learns to map inputs to outputs by minimising its mistakes.

  • Example: Predicting house prices. Input = size, location, number of rooms. Output = price (a number we already know for past sales).
  • Example: Email spam detection. Input = email text. Output = "spam" or "not spam".
  • Use when: You have labelled historical data and want to predict or classify new data.

2. Unsupervised Learning

The model is given unlabelled data and must find structure on its own — no correct answers are provided.

  • Example: Customer segmentation. Group thousands of customers by purchasing behaviour without being told what the groups should be.
  • Example: Anomaly detection. Find unusual transactions in a bank's data without knowing in advance what "unusual" looks like.
  • Use when: You want to discover hidden patterns or structure in your data.

3. Reinforcement Learning

An agent learns by interacting with an environment, receiving rewards for good actions and penalties for bad ones. Over thousands of trials, it learns an optimal strategy.

  • Example: Training a game-playing AI (like AlphaGo). The agent plays millions of games and learns which moves lead to winning.
  • Example: Self-driving car navigation — the car gets rewarded for smooth, safe driving and penalised for collisions.
  • Use when: The problem involves sequential decisions in a dynamic environment.

Why Python for AI/ML?

Python has become the dominant language for AI and ML for several reasons: it is readable, easy to learn, and has a rich ecosystem of libraries built specifically for data science and machine learning. The three you will use throughout this course are:

  • NumPy — fast numerical computation with arrays and matrices
  • Pandas — loading, cleaning, and analysing tabular data
  • Scikit-learn — ready-to-use ML algorithms (covered in Module 2)

Coding Basics: Variables, Lists, and Tables

Before you build AI, you need to understand how code stores information. A variable gives a name to a value. A list stores multiple values in order. A table stores rows and columns so you can compare examples.

AI and ML use those same basics at a larger scale. A model cannot learn from an idea in your head; it learns from data that your code loads, names, shapes, and passes into an algorithm.

Step 1 — NumPy: Fast Numerical Arrays

Python lists are slow for maths. NumPy arrays are fast. This step creates a 2-row, 3-column array (a matrix), then prints its shape, its mean, and the array doubled.

import numpy as np

matrix = np.array([[1, 2, 3],
                   [4, 5, 6]])

print("Shape:", matrix.shape)       # (2, 3) — 2 rows, 3 columns
print("Mean:", matrix.mean())       # 3.5 — average of all values
print("Doubled:\n", matrix * 2)     # element-wise multiplication

Step 2 — Pandas: Working With Tabular Data

Think of a DataFrame as a spreadsheet in Python. Each column is a feature; each row is one data point (observation). This step builds a small DataFrame, then inspects it three different ways.

import pandas as pd

data = {
    'name':       ['Alice', 'Bob', 'Carol', 'Dave'],
    'age':        [28, 34, 22, 45],
    'salary':     [52000, 78000, 41000, 95000],
    'department': ['Engineering', 'Marketing', 'Engineering', 'Finance']
}
df = pd.DataFrame(data)

print("\n--- Full DataFrame ---")
print(df)

print("\n--- Summary Statistics ---")
print(df[['age', 'salary']].describe())

print("\n--- Average salary by department ---")
print(df.groupby('department')['salary'].mean())

Understanding the Output

When you run this script you will see the DataFrame printed as a table, a statistical summary showing min/max/mean for numerical columns, and the average salary split by department. This kind of exploration — called Exploratory Data Analysis (EDA) — is the very first thing a data scientist or ML engineer does with any new dataset.

In every real ML project, you will: (1) load data into a DataFrame from a CSV, database, or API, (2) explore it with .describe(), .groupby(), and .head(), (3) clean it by handling missing values and fixing types, and (4) feed the processed arrays into an ML model. The ML Workflow: The Big Picture

Before jumping into algorithms, it is worth understanding the end-to-end process you will follow in every module:

  1. Define the problem — what are you predicting? Is it a number (regression) or a category (classification)?
  2. Collect and explore data — load it, look at it, understand its shape and quirks
  3. Preprocess — clean missing values, encode categories, scale numbers (Module 2)
  4. Choose and train a model — pick an algorithm, fit it to your training data
  5. Evaluate — measure performance on data the model has never seen
  6. Improve and iterate — tune, add features, try different algorithms
  7. Deploy — make the model available to real users (Module 6)
