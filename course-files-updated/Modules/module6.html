Reinforcement Learning

Reinforcement Learning (RL) is fundamentally different from supervised and unsupervised learning. There is no labelled dataset. Instead, an agent learns by taking actions in an environment and receiving rewards (positive) or penalties (negative) as feedback.

Over thousands — or millions — of trials, the agent learns a policy: a strategy that maps each situation (state) to the best action. This is how AlphaGo beat the world champion, how OpenAI Five beat professional Dota 2 players, and how robotic arms learn to pick up objects.

  • Agent — the learner/decision-maker (the AI)
  • Environment — the world the agent interacts with (a game, a robot's physical surroundings)
  • State — the agent's current observation of the environment
  • Action — what the agent can do (move left, press a button, turn a joint)
  • Reward — the signal that tells the agent if it did something good or bad
  • Policy — the strategy: given a state, what action to take

Reference: Q-Learning on CartPole

Requires the gymnasium package, which isn't available in the browser sandbox — run this in Colab or Jupyter.

# Install: pip install gymnasium
import gymnasium as gym
import numpy as np

# ── CartPole: balance a pole on a moving cart ─────────────────────────
# State: [cart position, cart velocity, pole angle, pole angular velocity]
# Actions: push cart LEFT (0) or RIGHT (1)
# Reward: +1 for every timestep the pole stays upright
# Episode ends when pole falls or cart goes out of bounds

env = gym.make('CartPole-v1')
print(f"State space:  {env.observation_space}")
print(f"Action space: {env.action_space}")  # Discrete(2) → 0 or 1

# ── Simple random policy (baseline) ──────────────────────────────────
# Before training, let's see how long a random agent survives.
total_rewards = []
for episode in range(20):
    state, _ = env.reset()
    total_reward = 0
    for step in range(500):
        action = env.action_space.sample()  # random action
        state, reward, terminated, truncated, _ = env.step(action)
        total_reward += reward
        if terminated or truncated:
            break
    total_rewards.append(total_reward)

print(f"\nRandom policy avg survival: {np.mean(total_rewards):.1f} steps")
print(f"(Max possible: 500 steps)")

# ── Q-Learning implementation ─────────────────────────────────────────
# Q-Learning learns a table: Q[state][action] = expected future reward
# Problem: CartPole has continuous states, so we discretise them.

def discretise(state, bins=10):
    """Convert continuous state to discrete bin indices."""
    bounds_low  = [-4.8, -4.0, -0.418, -4.0]
    bounds_high = [ 4.8,  4.0,  0.418,  4.0]
    indices = []
    for i, (val, lo, hi) in enumerate(zip(state, bounds_low, bounds_high)):
        idx = int((val - lo) / (hi - lo) * bins)
        idx = max(0, min(bins - 1, idx))
        indices.append(idx)
    return tuple(indices)

# Q-table: (10^4 states) × 2 actions
Q = np.zeros([10, 10, 10, 10, 2])

alpha   = 0.1    # learning rate: how much to update Q on each step
gamma   = 0.99   # discount factor: how much future rewards matter
epsilon = 1.0    # exploration rate: 1.0 = explore randomly at first

rewards_per_episode = []
n_episodes = 2000

for episode in range(n_episodes):
    state, _ = env.reset()
    s = discretise(state)
    total_reward = 0

    for step in range(500):
        # Epsilon-greedy: explore randomly or exploit best known action
        if np.random.rand() < epsilon:
            action = env.action_space.sample()       # explore
        else:
            action = np.argmax(Q[s])                  # exploit

        next_state, reward, terminated, truncated, _ = env.step(action)
        ns = discretise(next_state)

        # Q-learning update rule (the core of the algorithm)
        best_future = np.max(Q[ns])
        Q[s][action] += alpha * (reward + gamma * best_future - Q[s][action])

        s = ns
        total_reward += reward
        if terminated or truncated:
            break

    # Decay epsilon: over time, exploit more and explore less
    epsilon = max(0.01, epsilon * 0.995)
    rewards_per_episode.append(total_reward)

    if (episode + 1) % 500 == 0:
        avg = np.mean(rewards_per_episode[-100:])
        print(f"Episode {episode+1:4d} | Avg reward (last 100): {avg:.1f} | ε={epsilon:.3f}")

env.close()
print(f"\nFinal avg reward: {np.mean(rewards_per_episode[-100:]):.1f} / 500")

Generative AI

Generative AI creates new content — images, text, audio, video, code — rather than classifying or predicting from existing data. It is the fastest-moving area of AI and the one generating the most industry demand right now.

Generative Adversarial Networks (GANs)

A GAN trains two networks in opposition: a Generator that creates fake data, and a Discriminator that tries to tell real from fake. The Generator improves by fooling the Discriminator; the Discriminator improves by catching the Generator. After training, the Generator produces highly realistic outputs. GANs can generate photorealistic faces, artwork, and synthetic training data.

Large Language Models (LLMs)

LLMs like GPT-4 and Claude are Transformer-based models trained on vast amounts of text. They learn the statistical patterns of language so thoroughly that they can generate coherent, contextually appropriate text, answer questions, write code, and reason about complex topics. The key insight is that predicting the next word at scale on enough data produces emergent capabilities far beyond simple text completion.

Reference: Text Generation with GPT-2

Requires the transformers and torch libraries and downloads a ~500MB model — run this in Colab or Jupyter.

# Install: pip install transformers torch
from transformers import pipeline, set_seed

# ── Load GPT-2 text generation pipeline ──────────────────────────────
# GPT-2 is OpenAI's 2019 model (~500MB). Small by today's standards
# but still impressive and free to run locally.
generator = pipeline('text-generation', model='gpt2')
set_seed(42)  # for reproducible outputs

# ── Generate text from a prompt ───────────────────────────────────────
prompts = [
    "The future of artificial intelligence in healthcare will",
    "A machine learning engineer's typical day involves",
    "The most important skill for breaking into AI is"
]

for prompt in prompts:
    result = generator(
        prompt,
        max_new_tokens=60,
        num_return_sequences=1,
        temperature=0.8,     # 0=deterministic, 1=random, >1=very creative
        do_sample=True,
        pad_token_id=50256   # suppress warning
    )
    print(f"PROMPT: {prompt}")
    print(f"OUTPUT: {result[0]['generated_text']}")
    print("─" * 60)

# ── Key parameters explained ──────────────────────────────────────────
# temperature: controls randomness. Lower = more focused/repetitive.
# top_p: nucleus sampling — only consider tokens whose cumulative
#         probability exceeds top_p. Balances quality and diversity.
# max_new_tokens: how many tokens to generate beyond the prompt.
print("Text generation complete.")

MLOps: Taking Models to Production

Building a model that works in a Jupyter Notebook is one thing. MLOps (Machine Learning Operations) is the practice of reliably deploying, monitoring, and maintaining ML models in production — where real users depend on them 24/7.

Without MLOps, models degrade silently as the real world changes. With it, you have automated pipelines that retrain, test, and redeploy models continuously.

  • Model versioning — track experiments and model versions (tools: MLflow, Weights & Biases)
  • CI/CD for ML — automated testing and deployment pipelines
  • Model serving — expose the model via an API so applications can use it (tools: FastAPI, TensorFlow Serving, BentoML)
  • Monitoring — detect when model performance degrades due to data drift
  • Data pipelines — automate data collection, preprocessing, and retraining (tools: Airflow, Prefect)

Reference: Serving a Model as a REST API

Writes real files and expects a local uvicorn server — run this in your own environment, not the browser sandbox.

# Install: pip install fastapi uvicorn scikit-learn joblib
# This creates a production-ready API for your ML model.

# ── Step 1: Train and save the model (run once) ───────────────────────
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris
import joblib

X, y = load_iris(return_X_y=True)
model = RandomForestClassifier(n_estimators=100, random_state=42)
model.fit(X, y)
joblib.dump(model, 'iris_model.pkl')
print("Model saved to iris_model.pkl")

# ── Step 2: Create the API (save as api.py and run with uvicorn) ──────
api_code = '''
from fastapi import FastAPI
from pydantic import BaseModel
import joblib
import numpy as np

app = FastAPI(title="Iris Classifier API")
model = joblib.load("iris_model.pkl")
CLASS_NAMES = ["setosa", "versicolor", "virginica"]

class IrisFeatures(BaseModel):
    sepal_length: float
    sepal_width:  float
    petal_length: float
    petal_width:  float

@app.get("/health")
def health():
    return {"status": "ok"}

@app.post("/predict")
def predict(features: IrisFeatures):
    X = np.array([[
        features.sepal_length,
        features.sepal_width,
        features.petal_length,
        features.petal_width
    ]])
    prediction = model.predict(X)[0]
    probabilities = model.predict_proba(X)[0]
    return {
        "predicted_class": CLASS_NAMES[prediction],
        "confidence": float(probabilities[prediction]),
        "all_probabilities": {
            name: float(prob)
            for name, prob in zip(CLASS_NAMES, probabilities)
        }
    }
'''

with open('api.py', 'w') as f:
    f.write(api_code.strip())

print("API file saved. To run it:")
print("  uvicorn api:app --reload")
print()
print("Then test it with:")
print('  curl -X POST "http://localhost:8000/predict" \\')
print('       -H "Content-Type: application/json" \\')
print('       -d \'{"sepal_length":5.1,"sepal_width":3.5,')
print('             "petal_length":1.4,"petal_width":0.2}\'')
print()
print("Expected response:")
print('  {"predicted_class": "setosa", "confidence": 0.97, ...}')

Step 2 — Clean the Text & Vectorise With TF-IDF

Same cleaning pipeline as Module 4, then split into train/test sets and convert to TF-IDF vectors.

import re
from sklearn.model_selection import train_test_split
from sklearn.feature_extraction.text import TfidfVectorizer

def clean_text(text):
    text = text.lower()
    text = re.sub(r'[^a-z\s]', '', text)
    return text

texts_clean = [clean_text(t) for t in texts]

X_train, X_test, y_train, y_test, raw_train, raw_test = train_test_split(
    texts_clean, labels, texts, test_size=0.3, random_state=42
)

vectorizer = TfidfVectorizer(max_features=500, ngram_range=(1,2))
X_train_vec = vectorizer.fit_transform(X_train)
X_test_vec  = vectorizer.transform(X_test)
print(f"Vocabulary size: {len(vectorizer.vocabulary_)}")

Final Extension: Build Your Own Safe AI Agent

After completing the core capstone, you can design your own beginner AI agent. An AI agent is a program that receives input, uses rules or a model to reason about that input, and recommends or performs a bounded action.

The goal is to build something useful and safe: a study helper, task organizer, budgeting assistant, writing coach, customer-support triage tool, or research summarizer. Your agent should help a person make better decisions without hiding what it is doing.

Safe Agent Rules
  • Keep the scope narrow: define exactly what the agent is allowed to help with.
  • Keep a human in control: the agent can suggest actions, but a person should approve important decisions.
  • Protect privacy: do not collect passwords, private keys, banking logins, medical records, or unnecessary personal data.
  • Stay legal and non-harmful: do not build agents for hacking, surveillance, fraud, harassment, weaponization, or bypassing security systems.
  • Explain the output: the agent should show why it recommended something, not just give an answer.
